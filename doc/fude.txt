1. INTRODUCTION                                        *fude-introduction*

fude.nvim is a Neovim plugin that enables GitHub PR code review directly
in the editor. It provides:

- Base branch diff preview in a side pane
- Virtual text indicators for existing PR comments
- Create, view, reply, edit, and delete PR review comments
- Suggest code changes with GitHub's suggestion blocks
- Navigate between comments
- Browse changed files with Telescope or quickfix
- View PR overview with title, description, labels, reviewers, CI status, and comments
- Clickable GitHub references (#123) and URLs in floating windows
- @user, #issue, and _commit completion in comment windows (blink.cmp / nvim-cmp)
- Mark files as viewed/unviewed (synced with GitHub)
- Toggleable side panel showing review scope and changed files
- Gitsigns integration for diff base switching

==============================================================================
2. REQUIREMENTS                                        *fude-requirements*

- Neovim >= 0.10
- GitHub CLI (`gh`) installed and authenticated
- Git repository with a PR on the current branch

Optional:
- telescope.nvim for picker UI with diff preview (changed files and review scope)
- gitsigns.nvim for automatic diff base switching
- blink.cmp or nvim-cmp for @user / #issue / _commit completion

4. COMMANDS                                              *fude-commands*

:FudeReviewStart                                      *:FudeReviewStart*
    Start review session for the current branch's PR. Detects the PR
    automatically using `gh pr view`, fetches comments, sets up
    extmarks, and switches gitsigns base to the PR base branch.
    Does not open the diff preview (use |:FudeReviewDiff| for that).

:FudeReviewStop                                        *:FudeReviewStop*
    Stop review mode and clean up all preview windows, extmarks,
    autocommands, and restore original settings.

:FudeReviewToggle                                    *:FudeReviewToggle*
    Toggle review session on or off.

:FudeReviewLocal [{base}]                              *:FudeReviewLocal*
    Start local (pre-PR) review mode against {base} (default: the
    repository's default branch). Changed files come from the local
    git diff (merge-base with {base}, plus untracked files) and review
    comments are stored in `.fude/reviews/<session-id>.jsonl` inside
    the worktree instead of GitHub. Resumes the session recorded in
    `.fude/current.json` when one exists.

:FudeReviewLocalStop                               *:FudeReviewLocalStop*
    Stop local review mode and clear `.fude/current.json`. The
    session's JSONL file is kept for later reference.

:FudeReviewLocalScope [{scope}]                   *:FudeReviewLocalScope*
    Switch the local review scope (diff base). {scope} is `base` or
    `uncommitted`; with no argument, opens a picker. `base` (default)
    diffs against the merge-base with the base branch (whole branch);
    `uncommitted` diffs against HEAD (staged + unstaged changes only).
    Comments are unaffected — they anchor to the working tree.

:FudeReviewResolve                                   *:FudeReviewResolve*
    Toggle resolved status of the comment thread on the current line.
    Available in local review mode only. Resolved threads stay visible
    with a `[resolved]` badge.

:FudeReviewDiff                                        *:FudeReviewDiff*
    Toggle the diff preview window. Opens a side-by-side diff view
    showing the base branch version of the current file, or closes
    it if already open. Requires an active review session.

:FudeReviewComment                                  *:FudeReviewComment*
    Create a review comment on the current line (normal mode) or
    selected lines (visual mode). Opens a floating window for
    composing the comment. Press `<CR>` to save as GitHub pending review
    (visible on PR page with "Pending" status), `q` to cancel.
    Closing with `q` or `<Esc>` (where bound) when the buffer has unsaved
    changes prompts a 3-way choice: save the text as a local draft,
    discard it, or keep editing. Saved drafts are restored the next time
    you open input for the same location, and persist across PR switches
    and Neovim restarts. See |fude-drafts|.
    Pending comments can be submitted as a review using |:FudeReviewSubmit|.

:FudeReviewViewComment                          *:FudeReviewViewComment*
    View existing review comments on the current line in a floating
    window. Both submitted and pending review comments are shown.
    Press `r` to reply, `e` to edit, `d` to delete, `q` to close.
    Edit and delete are only available for your own comments.
    Editing opens a two-pane window (thread above, editable comment
    below) pre-filled with the original comment body.

:FudeReviewFiles                                      *:FudeReviewFiles*
    List files changed in the PR. Uses Telescope with diff preview
    by default, falls back to quickfix if Telescope is not available.
    Configurable via `file_list_mode`.

    Each file entry shows:
    - Viewed indicator (✓ when marked as viewed on GitHub)
    - File status (+/-/~/R/C)
    - Additions/deletions count
    - Comment count (💬N if comments exist, 💬N(outdated:M) if outdated)
    - File path

    Comment counts include both submitted and pending comments.
    Outdated comments (where the code has changed since the comment)
    are shown separately as `(outdated:N)`.
    When using Telescope, if pending comments exist, the count is
    highlighted with `DiagnosticHint`; otherwise `DiagnosticInfo`
    is used. In quickfix mode, comment counts are shown without
    these highlights.

:FudeReviewNextFile                                *:FudeReviewNextFile*
    Open the next changed file in the PR. Wraps around from the
    last file to the first. If the current buffer is not part of
    the PR's changed files, opens the first changed file.
    Equivalent Lua API: `require("fude.files").next_file()`.
    Requires an active review session.

:FudeReviewPrevFile                                *:FudeReviewPrevFile*
    Open the previous changed file in the PR. Wraps around from the
    first file to the last. If the current buffer is not part of
    the PR's changed files, opens the last changed file.
    Equivalent Lua API: `require("fude.files").prev_file()`.
    Requires an active review session.

:FudeReviewScope                                    *:FudeReviewScope*
    Select the review scope. Opens a picker showing the PR's commit
    history and a "Full PR" option. By default, the scope is the full
    PR (base...HEAD).

    Each commit entry shows its position as `[1/10]` to indicate
    order within the PR. The currently active scope is marked with
    a `▶` indicator.

    When a specific commit is selected, the plugin checks out that
    commit and switches the review context: |:FudeReviewFiles| shows
    only files changed in that commit, and the diff preview uses the
    commit's parent as the base. Gitsigns base is also updated.

    When "Full PR" is selected, the original HEAD is restored and
    the full PR scope is re-applied.

    When using Telescope, a preview pane shows the changed files
    for the selected scope with status icons and diff statistics
    (+additions / -deletions). For commit entries, file data is
    fetched asynchronously from the GitHub API and cached.

    Press `<Tab>` on a commit entry to toggle its reviewed state.
    Reviewed commits are shown with a checkmark indicator. This
    state is tracked locally within the session.

    Note: Uncommitted changes must be committed or stashed before
    switching to a commit scope.
    Requires an active review session.

:FudeReviewScopeNext                            *:FudeReviewScopeNext*
    Move to the next review scope. Cycles through:
    Full PR → commit 1 → commit 2 → ... → commit N → Full PR.
    Requires an active review session.

:FudeReviewScopePrev                            *:FudeReviewScopePrev*
    Move to the previous review scope. Cycles through:
    Full PR → commit N → ... → commit 2 → commit 1 → Full PR.
    Requires an active review session.

:FudeReviewOverview                                *:FudeReviewOverview*
    Show PR overview in a split-pane floating window. The left pane
    displays title, state, author, base/head branches, description,
    and issue-level comments. The right pane shows reviewers with
    review status, assignees, labels, and CI check status with
    pass/fail indicators and a summary count.
    Requires an active review session.
    Sections use Markdown headings (`##`) and are foldable with
    standard Neovim fold commands (`za`, `zc`, `zo`, `zM`, `zR`).
    Individual comments use `###` headings and can be folded
    independently. All folds are open by default.
    Section marks are set on the left pane: `'d` (description),
    `'c` (comments). Customizable via `overview.marks`.
    Press `C` to write a new comment, `R` to refresh,
    `<Tab>` to switch between panes, `]c`/`[c` to navigate
    review comments, `q` to close.

:FudeReviewSuggest                                  *:FudeReviewSuggest*
    Suggest a code change on the current line (normal mode) or
    selected lines (visual mode). Opens a floating window pre-filled
    with a GitHub suggestion block containing the selected code.
    Edit the code inside the block, then press `<CR>` to save as
    GitHub pending review. Pending comments can be submitted as a
    review using |:FudeReviewSubmit|. The suggestion appears on the PR
    with an "Apply suggestion" button.

:FudeReviewListComments                        *:FudeReviewListComments*
    Open a 3-pane floating comment browser showing all PR review
    comments and PR-level (issue) comments. Does not require
    Telescope.

    Left pane: comment list sorted by time (newest first).
      - Review comments: `date  @author  path:line`
      - PR-level comments: `date  PR Comment`
      - Pending comments: `date  [pending]  path:line`
    Right upper: thread display for the selected comment.
    Right lower: input area for reply/edit/new comment.
      For review comments, defaults to Reply mode.
      For PR-level comments, defaults to New PR Comment mode.

    Cursor movement in the left pane auto-updates the right side.

    Keymaps (left pane):~
      `<CR>`    Jump to file at comment line (review comments)
      `R`       Refresh comments
      `q`       Close browser
      `<Tab>`   Switch to thread pane

    Keymaps (upper pane):~
      `e`       Edit own comment
      `d`       Delete own comment (with confirmation)
      `q`       Close browser
      `<Tab>`   Switch to lower pane

    Keymaps (lower pane):~
      `<CR>`    Submit reply/edit/new comment
      `q`       Cancel and return to list
      `<Tab>`   Switch to list pane
      `<C-u>`   Scroll thread pane up
      `<C-d>`   Scroll thread pane down

    Requires an active review session.

:FudeReviewViewed                                    *:FudeReviewViewed*
    Mark the current file as viewed on GitHub. Uses the GitHub
    GraphQL API to sync the viewed state. The viewed status is
    reflected in the |:FudeReviewFiles| picker.
    Requires an active review session.

:FudeReviewUnviewed                                *:FudeReviewUnviewed*
    Unmark the current file as viewed on GitHub. Resets the
    viewed status for the current file.
    Requires an active review session.

:FudeOpenPRURL                                          *:FudeOpenPRURL*
    Open the PR in the browser. Uses the cached PR URL when a review
    session is active. Otherwise, detects the PR from the current
    branch via `gh` CLI.

:FudeCopyPRURL                                          *:FudeCopyPRURL*
    Copy the PR URL to the system clipboard (`+` register). Uses the
    cached PR URL when a review session is active. Otherwise, detects
    the PR from the current branch via `gh` CLI.

:FudeCreatePR                                            *:FudeCreatePR*
    Create a draft PR from a template. Searches for GitHub PR templates
    in standard locations (`.github/PULL_REQUEST_TEMPLATE/`,
    `PULL_REQUEST_TEMPLATE.md`, etc.). If multiple templates are found,
    shows a Telescope picker with preview (when Telescope is available),
    otherwise falls back to |vim.ui.select()| without a preview pane.
    If one template is found, uses it directly. If none, opens with an
    empty body.

    Opens a two-pane floating window:
    - Upper pane: PR title (single line)
    - Lower pane: PR body (pre-filled with template content)

    Press `<Tab>` to switch between title and body panes.
    Press `<CR>` to create a draft PR.
    Press `q` to cancel.

    Does not require an active review session. Works in any git
    repository with a branch that can create a PR.

:FudeEditPR                                              *:FudeEditPR*
    Edit the current PR's title and body. Opens a two-pane floating
    window similar to |:FudeCreatePR|:
    - Upper pane: PR title (editable)
    - Lower pane: PR body (editable, Markdown)

    Press `<Tab>` to switch between panes.
    Press `<CR>` to update the PR.
    Press `q` to cancel.

    When review mode is active, uses the PR from the current session.
    Otherwise, detects the PR from the current branch using `gh pr view`.

    Does not require an active review session.

:FudeReviewSubmit                                    *:FudeReviewSubmit*
    Submit all pending comments as a single GitHub review. This allows
    you to batch comments together instead of posting them individually.

    Workflow:
    1. Create comments using |:FudeReviewComment| (saved as GitHub pending)
    2. Comments appear on the PR page with "Pending" status
    3. Run |:FudeReviewSubmit| to submit all pending comments as one review
    4. Select review type: Comment, Approve, or Request Changes
    5. Optionally add a review body message
    6. All pending comments are submitted together

    If there are no pending comments, you can still submit a review
    with just a body (useful for Approve or Request Changes).

:FudeReviewReload                                  *:FudeReviewReload*
    Reload review data from GitHub. Fetches the latest comments,
    changed files, viewed file states, and PR commits in parallel.
    Useful when other reviewers have added comments or the PR has
    been updated. Can also be triggered automatically using the
    `auto_reload` configuration option.
    Requires an active review session.

:FudeReviewToggleCommentStyle            *:FudeReviewToggleCommentStyle*
    Toggle comment display style between "virtualText" and "inline".
    - "virtualText": Shows `#N` and `pending` indicators at the end of
      comment lines (original behavior).
    - "inline": Shows full comment content (author, timestamp, body)
      below the code line using virtual lines with a box border.
    The current style is stored in the session state and persists until
    review mode is stopped or explicitly toggled again.

    In inline mode, `]c` / `[c` navigation moves between comments
    without opening the floating comment viewer (since comments are
    already visible inline). The flash highlight still applies.

    When the cursor is on a comment line (in either virtualText or inline
    mode), a hint is displayed at the end of the line showing available
    actions (e.g., `<leader>ev: view/reply/edit/delete | <leader>eT: toggle
    comment style`). If keybindings for |:FudeReviewViewComment| or
    |:FudeReviewToggleCommentStyle| are detected, the hint shows those
    keybindings with `<leader>` notation. Otherwise, it shows the command
    names. The hint disappears when the cursor moves to a non-comment line.
    Requires an active review session.

:FudeReviewPanel                                  *:FudeReviewPanel*
    Toggle the review side panel. The panel is a sidebar split window
    that shows two sections:

    Review Scope:~
      In PR review, lists the PR's commit history with a "Full PR"
      option at the top; reviewed commits show a checkmark indicator.
      In local review (|fude-local-review|), lists the two local diff
      scopes ("Base branch" and "Uncommitted"); `<CR>` or `<Tab>`
      switches between them. The active scope is marked with `▶`.

    Files:~
      Lists all changed files in the current scope with status icons,
      additions/deletions counts, and viewed state. The header shows
      the viewed file count out of total (e.g., "Files (Reviewed: 3/10)").

    Keymaps (panel):~
      `<CR>`    Select scope entry or open file
      `<Tab>`   Toggle reviewed (scope) / viewed (file)
      `t`       Toggle files between flat list and tree
      `R`       Reload review data from GitHub
      `q`       Close panel

    When opening a file with `<CR>`, the cursor moves to a non-panel
    window before opening the file. The panel stays open.
    The panel auto-refreshes when the scope changes, data is reloaded,
    viewed/reviewed state is toggled, or the current buffer changes.
    The currently open file is marked with `▶` in the Files section,
    and the panel cursor follows the active file automatically.
    Configurable via `sidepanel` options. Requires an active session.

:FudeReviewToggleFileTree              *:FudeReviewToggleFileTree*
    Toggle the open side panel's Files section between flat list and
    directory tree. This changes only the current panel session; the
    default is controlled by `sidepanel.file_tree`.
    Requires the side panel to be open.

:FudeReviewToggleGitsigns              *:FudeReviewToggleGitsigns*
    Toggle gitsigns base between PR base and HEAD.
    By default during review mode, gitsigns uses the PR base (or commit
    parent when in commit scope) to show changes in the working tree.
    This command temporarily switches gitsigns to use HEAD as the base,
    showing uncommitted changes in the working tree.

    Execute again to restore the PR base display. Useful for checking
    your local uncommitted changes while in review mode.

    When switching review scope (|:FudeReviewScope|, |:FudeReviewScopeNext|,
    |:FudeReviewScopePrev|), the gitsigns display is automatically reset
    to PR base.

    Requires an active review session and gitsigns.nvim.

5. KEYMAPS                                                *fude-keymaps*

Comment navigation keymaps (]c / [c by default) are set as buffer-local
mappings during review mode and removed on stop. Other keymaps are not
set by the plugin. Configure them in your lazy.nvim spec or manually.
Suggested mappings:

  `<leader>et`  Toggle review mode
  `<leader>es`  Start review mode
  `<leader>eq`  Stop review mode
  `<leader>ec`  Create comment (normal/visual)  *
  `<leader>eS`  Suggest change (normal/visual) *
  `<leader>ev`  View comments on current line
  `<leader>er`  Reply to comment on current line
  `<leader>ef`  List changed files
  `<leader>eo`  PR overview
  `<leader>ed`  Toggle diff preview
  `<leader>eb`  Open PR in browser
  `<leader>ey`  Copy PR URL to clipboard
  `<leader>eC`  Select review scope (commit or full PR)
  `<leader>e]`  Next review scope
  `<leader>e[`  Previous review scope
  `<leader>ep`  Toggle review side panel
  `<leader>el`  List all review comments
  `<leader>eR`  Reload review data
  `<leader>em`  Mark current file as viewed
  `<leader>eM`  Unmark current file as viewed
  `]c`          Next comment
  `[c`          Previous comment
  `e`           Edit comment (in |:FudeReviewViewComment| float, own comments only)
  `d`           Delete comment (in |:FudeReviewViewComment| float, own comments only)
  `gx`          Open GitHub reference or URL under cursor (in floats)
  `<Tab>`       Toggle viewed state (in |:FudeReviewFiles| Telescope picker)
                Toggle commit reviewed state (in |:FudeReviewScope| picker)
                Switch between left/right panes (in |:FudeReviewOverview|)
  `'d`/`'c`    Jump to description/comments (overview left pane)
  `za`/`zc`/`zo` Toggle/close/open fold (overview sections and comments)
  `zM`/`zR`     Close/open all folds (in |:FudeReviewOverview|)

  * Visual mode mappings must use `:FudeReviewComment<CR>` (not `<cmd>`)
    to preserve the range. Using `<C-u>` will clear the range and
    result in single-line comments.

6. CONFIGURATION                                    *fude-configuration*

Default configuration: >lua
  require("fude").setup({
    -- Picker mode for changed files and review scope: "telescope", "quickfix", or "snacks"
    file_list_mode = "telescope",
    -- Diff filler character (nil to keep user's default)
    diff_filler_char = nil,
    -- Additional diffopt values applied during review
    diffopt = { "algorithm:histogram", "linematch:60", "indent-heuristic" },
    signs = {
      comment = "#",
      comment_hl = "DiagnosticInfo",
      pending = "⏳ pending",
      pending_hl = "DiagnosticHint",
      viewed = "✓",
      viewed_hl = "DiagnosticOk",
      draft = "✎ draft",
      draft_hl = "DiagnosticWarn",
    },
    float = {
      border = "single",
      -- Width/height as percentage of screen (1-100)
      width = 50,
      height = 50,
    },
    overview = {
      -- Width/height as percentage of screen (1-100)
      width = 80,
      height = 80,
      -- Right pane width as percentage of total overview width
      right_width = 30,
      -- Section marks (jump with 'd, 'c in left pane)
      marks = { description = "d", comments = "c" },
    },
    -- Flash highlight when navigating to a comment line (]c/[c)
    flash = {
      duration = 200, -- ms
      hl_group = "Visual",
    },
    -- Buffer-local keymaps set during review mode
    keymaps = {
      next_comment = "]c",
      prev_comment = "[c",
    },
    -- Comment display style: "virtualText" or "inline"
    comment_style = "virtualText",
    -- Inline display options (used when comment_style = "inline")
    inline = {
      show_author = true,
      show_timestamp = true,
      hl_group = "Comment",
      author_hl = "Title",
      timestamp_hl = "NonText",
      border_hl = "DiagnosticInfo",
    },
    -- Format file paths for display in UI (comment browser, file list, etc.)
    -- Function receives repo-relative path, returns formatted string.
    -- nil = display repo-relative path as-is (default).
    format_path = nil,
    -- strftime format for timestamps (system timezone)
    date_format = "%Y/%m/%d %H:%M",
    -- Auto-reload review data from GitHub
    auto_reload = {
      enabled = false,   -- Disabled by default
      interval = 30,     -- Seconds (minimum 10)
      notify = false,    -- Notify after auto-reload (true to show)
    },
    -- Side panel options
    sidepanel = {
      width = 40,       -- Panel width in columns (minimum 20)
      position = "left", -- "left" or "right"
      file_tree = "flat", -- "flat" or "tree"
    },
    -- Outdated comment display options
    outdated = {
      show = true,           -- Show outdated comments
      label = "[outdated]",  -- Label string
      hl_group = "Comment",  -- Highlight group for outdated label in comment browser
    },
  })
<

Options:

`file_list_mode`    "telescope" (default), "quickfix", or "snacks"
                  Controls which picker `:FudeReviewFiles` and
                  `:FudeReviewScope` use. "snacks" requires snacks.nvim;
                  it falls back to quickfix if not installed. For
                  `:FudeReviewScope`, "quickfix" falls back to
                  `vim.ui.select`.

`diff_filler_char`  Character for diff filler lines (default: nil).
                  Set to " " to hide the filler indicators.

`diffopt`           Table of diffopt values appended during review.
                  Set to nil to keep your existing diffopt settings.

`signs.comment`     Character shown in virtual text for comment indicators.

`signs.comment_hl`  Highlight group for comment indicators.

`signs.pending`     Text shown in virtual text for pending review comments.
                  Displayed alongside `signs.comment` when a line has both
                  submitted and pending comments (default: "⏳ pending").

`signs.pending_hl`  Highlight group for pending indicators
                  (default: "DiagnosticHint").

`signs.viewed`      Character shown in the file list for viewed files
                  (default: "✓").

`signs.viewed_hl`   Highlight group for viewed file indicators
                  (default: "DiagnosticOk").

`signs.draft`       Text shown in virtual text on lines that have an unsaved
                  local draft (see |fude-drafts|). For reply / edit drafts the
                  indicator appears on the targeted comment's line
                  (default: "✎ draft").

`signs.draft_hl`    Highlight group for draft indicators
                  (default: "DiagnosticWarn").

`float.border`      Border style for floating windows.

`float.width`       Floating window width as % of screen (default: 50).

`float.height`      Floating window height as % of screen (default: 50).

`overview.width`    Total overview width as % of screen (default: 80).
                  This is split between the left and right panes.

`overview.height`   Overview window height as % of screen (default: 80).

`overview.right_width`
                  Right pane width as % of total overview width
                  (default: 30). Minimum effective width is 15 columns.

`overview.marks`    Table mapping section names to mark characters
                  for the left pane.
                  Sections: "description", "comments".
                  Use `'d`, `'c` to jump to each section.
                  (default: { description = "d",
                  comments = "c" }).

`flash.duration`    Duration in milliseconds for the flash highlight
                  when navigating to a comment line (default: 200).

`flash.hl_group`    Highlight group for the flash effect (default: "Visual").

`keymaps.next_comment`
                  Buffer-local mapping for next-comment navigation, set
                  during review mode and removed on stop (default: "]c").

`keymaps.prev_comment`
                  Buffer-local mapping for previous-comment navigation,
                  set during review mode and removed on stop (default: "[c").

`auto_view_comment` When true, automatically opens the comment viewer
                  float after navigating to a comment line via ]c, [c,
                  or FudeReviewListComments (default: true).

`comment_style`     Comment display style: "virtualText" or "inline"
                  (default: "virtualText").
                  - "virtualText": Shows indicators (`#N`, `pending`)
                    at the end of lines.
                  - "inline": Shows full comment content below lines
                    using virtual lines.
                  Can be toggled at runtime with |:FudeReviewToggleCommentStyle|.

`inline.show_author`
                  Show author name in inline display (default: true).

`inline.show_timestamp`
                  Show timestamp in inline display (default: true).

`inline.hl_group`   Highlight group for comment body text in inline mode
                  (default: "Comment").

`inline.author_hl`  Highlight group for author name in inline mode
                  (default: "Title").

`inline.timestamp_hl`
                  Highlight group for timestamp in inline mode
                  (default: "NonText").

`inline.border_hl`  Highlight group for comment box border in inline mode
                  (default: "DiagnosticInfo").

`inline.markdown_highlight`
                  Enable Markdown syntax highlighting in inline comments
                  using tree-sitter markdown_inline parser. Highlights
                  **bold**, *italic*, `code`, and [link](url) syntax
                  (default: true). Requires tree-sitter markdown_inline
                  parser. Falls back to plain highlighting if unavailable.
                  Code blocks (```) are not highlighted.

`inline.markdown_hl`
                  Table of highlight groups for Markdown elements:
                  - `bold`: Highlight for **bold** text
                    (default: "@markup.strong")
                  - `italic`: Highlight for *italic* text
                    (default: "@markup.italic")
                  - `code`: Highlight for `inline code`
                    (default: "@markup.raw")
                  - `link`: Highlight for [link text]
                    (default: "@markup.link")
                  - `link_url`: Highlight for (url)
                    (default: "@markup.link.url")

`format_path`       Function to format file paths for display in the UI
                  (default: nil). When nil, repo-relative paths are shown
                  as-is (e.g. "lua/fude/init.lua"). Applied to:
                  - Comment browser left pane (path:line)
                  - Telescope/quickfix file list
                  - Comment picker
                  - Scope preview changed files
                  The function receives a repo-relative path and should
                  return a formatted string. Searching/filtering still
                  uses the full path. >lua
                    require("fude").setup({
                      format_path = function(path)
                        -- Show only filename
                        return vim.fn.fnamemodify(path, ":t")
                      end,
                    })
<

`date_format`       strftime format for timestamps (default: "%Y/%m/%d %H:%M").
                  Timestamps from GitHub (UTC) are converted to the system
                  timezone before formatting.

`sidepanel.width`   Width of the side panel in columns (default: 40).
                  Minimum effective width is 20 columns.

`sidepanel.position`
                  Side of the editor to open the panel: "left" or "right"
                  (default: "left").

`sidepanel.file_tree`
                  Files section layout for the side panel: "flat" or "tree"
                  (default: "flat"). The tree layout groups changed files
                  by directory and can be toggled for the current panel with
                  |:FudeReviewToggleFileTree| or `t` in the panel.

`outdated.show`     Show outdated comments (default: true).
                  Outdated comments are those whose position in the code
                  can no longer be tracked due to subsequent changes.

`outdated.label`    Label text shown for outdated comments
                  (default: "[outdated]").

`outdated.hl_group` Highlight group for outdated indicator in virtualText mode
                  (default: "Comment").

`auto_reload.enabled`
                  Enable automatic periodic reload of review data
                  (default: false). When enabled, comments, changed files,
                  viewed states, and PR commits are re-fetched at regular
                  intervals.

`auto_reload.interval`
                  Interval in seconds between auto-reloads (default: 30).
                  Minimum value is 10 seconds (values below 10 are clamped).

`auto_reload.notify`
                  Show a notification message after each auto-reload
                  completes (default: false). Manual `:FudeReviewReload`
                  always shows a notification regardless of this setting.

`on_review_start`   Callback function invoked after |:FudeReviewStart|
                  completes all data fetching (changed files, commits,
                  viewed files, comments, authenticated user).
                  Receives a table with PR information:
                  `{ pr_number, base_ref, head_ref, pr_url }`
                  (default: nil). >lua
                    require("fude").setup({
                      on_review_start = function(info)
                        vim.notify("Review started: PR #" .. info.pr_number)
                      end,
                    })
<

`drafts.enabled`    Enable local on-disk drafts for in-progress comment
                  input (default: true). When false, closing a dirty
                  comment buffer falls back to a plain Yes/No discard
                  confirmation and no drafts are saved or restored.
                  See |fude-drafts|.

`drafts.retention_days`
                  Drafts older than this many days are pruned when the
                  draft store is next loaded (default: 30). Set to 0 or a
                  negative number to keep drafts indefinitely.

7. STATUSLINE                                        *fude-statusline*

fude.nvim provides a function to display the current review scope in
your statusline.

`require("fude.scope").statusline()`
    Returns a string representing the current scope:
    - `"Scope: PR"` when reviewing the full PR
    - `"Scope: 3/10"` when reviewing commit 3 of 10
    - `""` (empty string) when fude.nvim is not active

Example with lualine: >lua
  lualine_x = {
    {
      function() return require("fude.scope").statusline() end,
      cond = function() return require("fude").is_active() end,
    },
  }
<

8. COMPLETION                                          *fude-completion*

Comment input windows (`FudeReviewComment`, `FudeReviewSuggest`) support GitHub
completion for `@user` mentions, `#issue/PR` references, and `_commit` hashes.

blink.cmp: >lua
  sources = {
    default = { "lsp", "path", "buffer", "snippets", "fude" },
    providers = {
      fude = {
        name = "fude",
        module = "fude.completion.blink",
        score_offset = 50,
        async = true,
      },
    },
  }
<

nvim-cmp: >lua
  require("cmp").register_source("fude",
    require("fude.completion.cmp").new())
  -- Add { name = "fude" } to your sources
<

Trigger characters:
  `@`  - GitHub collaborators (@user mentions)
  `#`  - Issues and PRs (#123 references)
  `_`  - PR commit hashes (filtered by [n/m] index and message)

Data for `@` and `#` is cached for 5 minutes per session and fetched from the
GitHub API asynchronously. Data for `_` uses locally cached PR commits (no
additional API call).

9. DRAFTS                                                    *fude-drafts*

When you close a comment input that has unsaved changes (via `q` / `<Esc>`),
fude.nvim offers to save the text as a *local draft* instead of discarding it:

    Save draft & close   Persist the current text locally and close.
    Discard & close      Throw the text away (and delete any saved draft).
    Keep editing         Return to the input without closing.

Drafts are local to your machine and are NOT sent to GitHub (unlike a pending
review comment, which is created by pressing `<CR>`). A saved draft is restored
automatically the next time you open input for the same target.

Drafts cover every comment input: line and range comments, |:FudeReviewSuggest|
suggestions, PR-level (issue) comments, replies, and edits. Each is keyed by
repository, PR number, and target, so drafts for different locations and
different PRs are kept independently and never collide. A draft is removed once
its comment is submitted, or when you explicitly discard it.

Lines that have a saved draft show a `draft` indicator in the diff (configurable
via `signs.draft` / `signs.draft_hl`), similar to the `pending` indicator. Line
and suggestion drafts mark their own line; reply and edit drafts mark the line of
the comment they target. PR-level (issue) drafts have no diff line and so are not
indicated in the buffer.

Drafts also appear in the comment browser (|:FudeReviewListComments|). A draft
that targets an existing comment/thread adds a `✎draft` marker to that entry; a
new draft with no submitted counterpart shows as a `[draft]` row. On a draft row,
`<CR>` jumps to the file (line/suggestion drafts) so you can resume editing
there, or focuses the input pane (PR-level drafts).

Drafts are stored as JSON at: >
    stdpath("state")/fude/drafts.json
<
and therefore survive switching between PRs and restarting Neovim. Stale drafts
are pruned on load (see `drafts.retention_days`). Set `drafts.enabled = false`
to turn the feature off entirely (see |fude-configuration|).

Limitations:
  - A draft is matched by line range; if the diff shifts the lines, the draft
    will not be restored at the new position (same as pending comments).
  - The draft file is written with last-write-wins semantics, so running
    multiple Neovim instances against the same draft store may drop a draft.

10. LOCAL REVIEW                                      *fude-local-review*

|:FudeReviewLocal| reviews the working tree before a PR exists — typically
to review AI-agent-generated code locally. No GitHub interaction happens in
this mode.

Data model~

Comments live in an append-only JSONL event log inside the worktree: >
    .fude/reviews/<session-id>.jsonl    one JSON event per line
    .fude/current.json                  pointer to the active session
<
Add `.fude/` to your `.gitignore`. Event kinds: `session` (header),
`comment` (thread root with path/start_line/end_line/body/author/
author_type/context), `reply`, `edit`, `move` (line re-anchor), `resolve`,
`reopen`, and `delete` (hides the comment from the view; the log line
remains as an audit trail).

Behavior~

  - Changed files come from the local git diff plus untracked files. The
    diff base depends on the scope (|:FudeReviewLocalScope|): `base`
    (default) uses the merge-base with the base ref (whole branch diff),
    `uncommitted` uses HEAD (staged + unstaged changes only). The
    statusline shows `Local: <base>` or `Local: uncommitted`.
  - Comments are saved immediately on `<CR>` — there is no pending review
    or submit step. |:FudeReviewSubmit| is not available in this mode.
  - Comment positions follow buffer edits via extmarks and are re-anchored
    in the JSONL on `:write` (as `move` events). Comments whose file or
    line disappeared are shown as `[outdated]` in the comment browser.
  - |:FudeReviewResolve| toggles a thread's resolved state (`[resolved]`
    badge). Resolved threads stay visible.
  - Comment drafts (|fude-drafts|) are disabled in local review mode
    (comments are saved locally anyway).
  - The statusline component (|fude-statusline|) shows `Local: <base>`.

AI agent integration~

The JSONL file is the only contract. An agent reads the events and appends
replies with `author_type: "agent"` (shown with an `[agent]` badge). Enable
`auto_reload` (see |fude-configuration|) to pick up agent replies
automatically. Agents must append whole lines only — never rewrite existing
lines. A skill scaffold for a resident Claude Code session is provided at
`contrib/skills/fude-watch/` in the repository.
